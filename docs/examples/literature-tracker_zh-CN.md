# 示例岗位：文献追踪员

本文以「文献追踪员」岗位为例，展示如何在 LinkWork 中配置一个完整的 AI 员工，让它持续追踪前沿动态、自动收集归档知识、定期生成报告。

---

## 岗位概述

| 属性 | 值 |
|------|-----|
| 岗位名称 | 文献追踪员 |
| 职责 | 追踪前沿动态，自动收集、归档外部知识，定期生成结构化报告 |
| 适用人群 | 研究人员、技术管理者、知识工作者 |
| 核心价值 | 7x24 在线的个人知识助理，信息降噪和知识沉淀 |

---

## 工作模式

### 模式一：按兴趣建立个人知识库

用户告诉文献追踪员自己关注的领域，AI 自动创建一套结构化的知识目录。每个人的知识库完全独立、互不可见。

以「AI 大模型」领域为例，生成的目录结构如下：

```
literature/
├── README.md                      # 个人知识库使用说明
├── daily-news/                    # 每日新闻速递（按日归档）
├── papers/                        # 论文速览 + 精读笔记
├── github-projects/               # GitHub 开源项目周报
├── reading-notes/                 # 深度阅读笔记（长期保留）
├── industry-reports/              # 行业报告与白皮书摘要
├── tech-blogs/                    # 优质技术博客文章评注
├── _templates/                    # 各类文档模板
└── _archive/                      # 过期内容自动归档区
```

目录结构并非固定，AI 会根据用户的兴趣领域（AI、金融、生物医药……）量身定制。

### 模式二：随手投喂，自动归档

日常看到好文章、好论文，只需把链接扔给文献追踪员。AI 会通过 Tavily 搜索抓取相关信息、提取摘要和核心观点、归档到知识库对应目录、纳入个人知识索引。

**示例交互：**

```
用户：https://arxiv.org/abs/2401.12345

AI：已抓取论文「Scaling Laws for Neural Language Models」
    → 归档至 papers/2026-03-05-scaling-laws.md
    摘要：本文提出……核心发现：① …… ② …… ③ ……
    已加入您的知识索引。
```

### 模式三：定时抓取，每日速递

配置好关注源后，文献追踪员每天自动运行：

- **早间**：通过 Tavily 搜索抓取最新论文动态、AI 新闻、GitHub Trending 项目
- **整理**：过滤噪声，按关注领域筛选，生成当日速递
- **归档**：写入 `daily-news/` 目录，同步更新知识索引

**示例输出：**

```
[每天早上 8:00 自动执行]

今日 AI 速递已生成 → daily-news/2026-03-05.md

  重点关注
  1. DeepSeek-V4 正式发布，推理性能提升 40%
  2. Google 开源 Gemma 3 系列模型

  新论文 (共 7 篇与您的关注匹配)
  1. "Attention is All You Need... Again" — Transformer 架构新变体
  2. ……

  GitHub 热门
  1. openai/swarm — 多 Agent 编排框架 (本周新增)
  2. ……
```

### 模式四：周期性报告，沉淀洞察

文献追踪员定期对积累的文献进行聚合分析，自动输出报告：

- **周报**：本周收录了什么、热点趋势是什么、值得深读的是哪几篇
- **月报**：本月领域全景、关键技术演进脉络、推荐阅读清单

### 模式五：随时提问，知识即用

所有归档的文献都进入了个人知识索引。用户可以随时用自然语言提问，AI 基于已积累的知识库进行回答：

```
用户：最近一个月关于 MoE 架构有哪些新进展？

AI：基于您知识库中的 12 篇相关文献，核心进展如下：
    1. DeepSeek 提出了「Fine-Grained Expert Segmentation」方案……
    2. Google 在 Gemini 2.5 中采用了动态路由策略……
    3. 清华团队的 OpenMoE 开源了完整训练代码……

    详细汇总已生成 → reading-notes/moe-summary-202603.md
```

## 岗位配置参考

### Skills 配置

| Skill | 用途 |
|-------|------|
| 文献抓取 | 从 URL 抓取全文，支持论文、博客、新闻站点 |
| 摘要提取 | 按模板提取核心观点、关键数据 |
| 知识索引 | 维护个人知识索引，支持语义检索 |
| 报告生成 | 按周/月自动生成趋势分析报告 |

### MCP 工具

| 工具 | 用途 |
|------|------|
| Tavily Web Search | 搜索论文、新闻、GitHub 项目等公开信息，支持 AI 优化的结构化搜索结果 |
| 网页抓取工具 | 访问和解析网页内容，提取正文 |

### 定时排班

| 任务 | Cron 表达式 | 说明 |
|------|-----------|------|
| 每日速递 | `0 8 * * *` | 每天早 8 点执行 |
| 周报生成 | `0 9 * * 1` | 每周一早 9 点执行 |
| 月报生成 | `0 10 1 * *` | 每月 1 号早 10 点执行 |

## 关键特性

| 特性 | 说明 |
|------|------|
| 个人专属 | 每人独立的知识库和记忆空间，互不可见 |
| 随用随喂 | 扔个链接即可，AI 自动抓取、整理、归档 |
| 自动运转 | 定时抓取 + 定期报告，无需人工干预 |
| 越用越聪明 | 积累越多、检索越准，报告越有深度 |
| 灵活定制 | 关注领域、抓取源、报告频率均可按需调整 |
