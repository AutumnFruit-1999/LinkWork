# Skills 系统

Skills 是 LinkWork 中 AI 员工的**可装卸能力模块**。每个 Skill 是一个独立的、声明式定义的能力单元，可以按需组合安装到不同岗位。

---

## 设计思想

传统 Agent 的能力边界模糊 — 装了什么插件、有什么能力，运行时才知道，而且 AI 可以自行安装和修改。

LinkWork 的做法不同：

- Skills **声明式定义** — 每个 Skill 有明确的名称、描述、版本和依赖
- Skills **构建时锁定** — 岗位构建时将 Skills 注入镜像，运行时只读
- Skills **管理员管控** — 哪些岗位可用哪些 Skill，由管理员决定，AI 不能自行安装

> 像给手机安装 App 一样给 AI 员工装配能力。管理员是应用商店的审核者，AI 员工是用户。

---

## 三层能力架构

```
岗位 (Role)
  └── Skills（能力模块）
        └── MCP 工具（外部工具接入）
```

**岗位**定义「AI 员工是谁」，**Skills** 定义「AI 员工能做什么」，**MCP 工具**定义「AI 员工能用什么工具」。三层解耦、自由组合、权限可控。

---

## Skill 的组成

一个 Skill 包含以下核心要素：

| 要素 | 说明 |
|------|------|
| 名称与描述 | Skill 的标识和功能说明 |
| 版本号 | 语义化版本（SemVer），支持版本回退 |
| 知识内容 | Markdown 格式的知识文件，注入 AI 的上下文 |
| 依赖声明 | 该 Skill 依赖的其他 Skills 或工具 |
| 适用范围 | 标签和分类，便于市场检索 |

### Skill 知识注入

Skill 的核心价值在于**知识注入**。每个 Skill 本质上是一组结构化的知识文件（Markdown），在任务执行时自动加载到 AI 的上下文中：

```
skills/
├── code-review/
│   ├── SKILL.md           # Skill 定义与元数据
│   ├── guidelines.md      # 代码审查指南
│   └── checklist.md       # 审查清单
└── data-analysis/
    ├── SKILL.md
    ├── methodology.md     # 分析方法论
    └── templates.md       # 报告模板
```

AI 员工在执行任务时，会自动获得当前岗位所有 Skills 的知识，无需额外配置。

## Skills 市场

LinkWork 提供内置的 Skills 市场，支持团队间共享和复用：

### 市场功能

| 功能 | 说明 |
|------|------|
| 发布 | 将个人验证过的 Skill 发布到团队市场 |
| 发现 | 按分类、标签、热度检索可用 Skills |
| 版本管理 | 每个 Skill 独立版本管理，支持回退 |
| 使用统计 | 查看每个 Skill 的调用次数和成功率 |
| 团队共享 | 设置 Skill 的可见范围（个人/团队/全局） |

### 生命周期

```
编写 Skill → 个人验证 → 发布到市场 → 管理员审核 → 团队可用 → 持续迭代
```

> 你在个人工具上打磨好的 Skills，可以直接放进 LinkWork，变成整个团队都能用的标准化能力。

## 构建时锁定

Skills 在岗位构建阶段被注入到容器镜像中：

1. 按岗位配置拉取对应版本的 Skills
2. 锁定精确版本号，写入镜像
3. 运行时只读，不可修改

这意味着：
- 同一个岗位的所有实例，Skills 配置完全一致
- Skills 变更必须重新构建镜像，不会出现"部分实例更新了、部分没更新"的情况
- 构建记录完整可追溯，任何时间点都能复现当时的 Skills 配置

> 详见 [Harness Engineering](./harness-engineering_zh-CN.md)。

## 常见 Skills 示例

| Skill | 说明 | 适用岗位 |
|-------|------|---------|
| 代码审查 | 代码质量、安全漏洞、性能问题审查 | 开发工程师 |
| 数据分析 | 数据清洗、统计分析、可视化报告 | 数据分析师 |
| 文档撰写 | 技术文档、API 文档、用户手册 | 技术写作 |
| 文献追踪 | 论文抓取、摘要提取、趋势分析 | 研究助理 |
| 运维巡检 | 服务健康检查、日志分析、告警处理 | 运维工程师 |
