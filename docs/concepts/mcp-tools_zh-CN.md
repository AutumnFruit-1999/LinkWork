# MCP 工具

LinkWork 通过 **MCP 工具总线**为 AI 员工接入外部能力。兼容 [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) 标准，所有外部工具通过统一的代理层接入，自动完成鉴权、路由和用量统计。

---

## 设计思想

AI 员工要完成实际工作，仅靠 LLM 推理是不够的 — 还需要调用数据库、访问 API、操作文件系统、控制浏览器等外部能力。

LinkWork 选择了 MCP 作为工具接入标准：

- **协议标准化** — 遵循 MCP 开放标准，生态兼容性好
- **统一代理** — 所有工具调用经过网关代理，不直连外部服务
- **鉴权管控** — 工具级别的权限控制，管理员决定谁能用什么工具
- **用量可观测** — 每次工具调用都有记录，支持计量和审计

---

## 三层工具架构

```
linkwork-mcp-gateway（工具网关）
  ├── 鉴权与路由
  ├── 健康检查
  └── 用量统计
        │
        ▼
  MCP 工具服务
  ├── 数据库查询工具
  ├── API 调用工具
  ├── 文件操作工具
  ├── 浏览器控制工具
  └── ...更多工具
```

### linkwork-mcp-gateway

MCP 工具网关是所有工具调用的唯一入口，负责：

| 功能 | 说明 |
|------|------|
| 工具注册与发现 | 管理所有已注册的 MCP 工具服务 |
| 鉴权代理 | 按岗位权限控制工具访问 |
| 健康检查 | 定时探活，自动下线不健康的工具服务 |
| 用量统计 | 记录每次工具调用的耗时、结果和费用 |
| 限流保护 | 按服务配置 QPS 限制，防止滥用 |

---

## 工具类型

LinkWork 当前支持以下类型的 MCP 工具服务：

| 类型 | 说明 | 示例 |
|------|------|------|
| HTTP | 远程 HTTP API 服务 | Jira、GitLab、数据库查询 |
| SSE | 远程 Server-Sent Events 服务 | 实时监控、流式数据源 |

> 更多类型（如 stdio 本地进程）将在后续版本中支持。

---

## 工具生命周期

### 注册

管理员在管理界面注册 MCP 工具服务，提供：

- 服务名称和描述
- 服务类型（HTTP / SSE）
- 服务地址
- 鉴权配置（API Key、Token 等）
- 健康检查地址（可选）

### 健康检查

注册后，平台自动进行定时健康检查：

| 状态 | 含义 |
|------|------|
| 在线 | 最近探活成功，响应正常 |
| 降级 | 响应延迟较高或间歇性失败 |
| 离线 | 连续多次探活失败 |

### 岗位关联

将工具分配给特定岗位：

- 一个岗位可以关联多个 MCP 工具
- 工具配置在岗位构建时固化到镜像
- 运行时 AI 员工只能使用已授权的工具

---

## MCP 工厂

MCP 工厂是 AI 供应链的一部分，提供工具的统一管理和治理：

- **工具注册与发现** — 统一的工具目录，按分类和标签检索
- **版本管理** — 工具服务的版本追踪
- **健康监控** — 实时感知所有工具服务的可用性
- **使用分析** — 调用量、延迟、错误率等统计数据

---

## 安全考虑

MCP 工具的接入遵循 LinkWork 的安全原则：

- **最小权限** — 每个岗位只能访问已授权的工具
- **凭证隔离** — 工具的鉴权凭证由平台管理，AI 员工不可见
- **网络管控** — 工具调用通过网关代理，不允许 AI 直连外部服务
- **审计追踪** — 每次工具调用都有完整的审计记录
